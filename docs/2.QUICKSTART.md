# 🚀 Quick Start

This guide covers all methods to configure MyAstroBoard.

## Prerequisites

### Required
- Running installation of MyAstroBoard
- Logged in the web dashboard

> [!NOTE]
> On first connection, credentials are `admin` / `admin`; you must change them.

## 📖 Configuration Guide

### 1. Configure Your Location

In **Parameters → Locations** (the guided setup wizard fills this in for you on first run):
- Enter your observatory name, no real influence only displayed on certain cards
- Set latitude and longitude:
  - **Decimal format**: 48.6434 (recommended for copy-paste)
   - **DMS format**: 48d38m36.16s or 48°38'36.16" (automatically converted)
  - Converted value shown below input for verification
- Set elevation in meters
- **Select timezone** from dropdown (IANA timezone list)

> [!TIP]
> Since v1.2 you can create up to **5 location presets** (backyard, club field, dark-sky site…), attribute them to users, and switch the active one from the sky status widget. See [LOCATIONS.md](LOCATIONS.md).

### 2. Configure Advanced Settings (Optional)

Expand the **Advanced Settings** section for calculations better tailored to your environment and equipment.

**Constraints** - these filters are always active and determine which targets SkyTonight considers observable:

| Constraint | Default | Role |
|---|---|---|
| **Altitude min** | 30° | Minimum elevation above the horizon. Below this, atmospheric distortion and light extinction degrade image quality significantly. |
| **Altitude max** | 80° | Maximum elevation (near-zenith limit). Avoids the blind spot many mounts have when pointing straight up. |
| **Airmass limit** | 2 | Thickness of atmosphere the light travels through (airmass 2 ≈ 30° elevation). Acts as a secondary guard alongside the altitude minimum. |
| **Object size min** | 0 arcmin | Allows compact targets (for example planetary nebulae) to remain selectable. Increase this value only if you want to hide very small objects. |
| **Object size max** | 300 arcmin | Filters out objects larger than 5° that would exceed a typical imaging field of view. |
| **Moon separation** | 45° | Minimum angular distance from the Moon. When **Moon illumination mode** is enabled (default), this value is overridden by the Moon's illumination percentage mapped 1:1 to degrees - e.g. a 70% Moon requires 70° of separation. |
| **Observable fraction** | 0.5 | A deep-sky object must satisfy all constraints for **at least 50%** of the nautical night to appear in results. Solar-system bodies use a relaxed threshold of 5% (~22 min) since a planet visible for a few hours is still a prime target. |

> [!NOTE]
> SkyTonight execution is scheduled. If you make a change and want to test immediately, click the "Run SkyTonight Now" button on the Parameters page.

## 📖 Usage Guide

### 1. Forecast

> [!NOTE]
> Forecast and astronomy calculations can be resource-intensive on first run. Caches are then reused with per-job TTLs (for example weather 1h, moon planner/report 2h, sun report 6h).

#### a. Weather

The Weather tab shows key information for the next 12 hours. This data comes from Open-Meteo, with some values calculated locally (such as AstroScore).

#### b. Best Window

Best Window estimates the best observing time window during the night. It is computed using the 3 modes explained in the tab.

#### c. Observation conditions

Charts provide a quick overview of upcoming weather conditions.

#### d. Moon & Sun

Useful information about the Moon and Sun.

### 2. SkyTonight

> [!TIP]
> Catalogue tabs are displayed after the first successful execution.

Navigate to the **SkyTonight** section to view SkyTonight outputs:

- **Sky Map**: Interactive polar sky map showing all observable targets at a glance
  - Filter by category (DSO, Bodies, Comets) and minimum AstroScore
  - Click any target on the map to highlight it in the table
- **Deep-Sky Target Table**: Sortable/filterable table with columns: ID, Name, Size, Foto Score, Mag, Constellation, Type, Altitude, Azimuth
  - Filter by foto score (e.g., show only ≥ 0.8 quality targets)
  - Filter by constellation or object type
  - Use the search box to find specific targets
  - Sort by any column
  - Click target name to view the altitude-vs-time plot in a popup
  - Click **➕ More** to see additional data (catalogue names, transit times, RA/Dec)
  - **Add to Astrodex**: Click the "➕ Add" button - objects already in your Astrodex show "✓ Captured"
  - **Plan My Night**: Click the "🌙 Add" button to add the target to tonight's observation plan - planned targets show "✓ Planned"
- **Bodies / Comets**: Separate tables (when enabled) with name, altitude, azimuth, visual magnitude, and the same Astrodex / Plan My Night actions

### 3. Plan My Night

> [!NOTE]
> Plan My Night is personal per user. `read-only` accounts can view but cannot edit plans.

The **Plan My Night** tab lets you build a private target timeline for a single observing night.

#### Adding Targets

- From any SkyTonight table (deep-sky, bodies, comets), click the **🌙 Add** button in the **Plan My Night** column.
- If no plan exists yet, the first add creates it automatically for the current night.

#### Plan States

| State | Meaning |
|---|---|
| **None** | No plan stored - first add will create one. |
| **Current** | Tonight's plan; targets can be added and edited freely. |
| **Previous** | Plan is from a past night and is locked. Clear it before starting a new one. |

#### Managing the Plan

- Set a **planned duration** per target (`HH:MM`).
- **Reorder** targets up/down within the timeline.
- **Mark done / undo** to track observation progress.
- **Remove** individual targets.
- **Add to Astrodex** directly from the plan without going back to the catalogue.
- A **timeline progress bar** and current-target banner are shown when you are within the planned night timeframe.

#### Exports

- **CSV**: Click the export button or navigate to `GET /api/plan-my-night/export.csv`
- **PDF**: `GET /api/plan-my-night/export.pdf`

### 4. Astrodex - Gotta Catch 'Em All!

> [!NOTE]
> Astrodex is your personal collection of celestial objects you've photographed. Each user has their own separate Astrodex.

The **Astrodex** tab provides a Pokedex-style collection system for astrophotography:

#### Adding Objects to Your Astrodex

There are two ways to add objects:

1. **From Catalogues**: 
   - Browse any SkyTonight catalogue (Messier, NGC, etc.)
   - Click the "➕ Add" button next to any object
   - Objects already in your Astrodex show "✓ Captured"

2. **Manually**:
   - Click "➕ Add Object" button in the Astrodex tab
   - Enter object name, type, constellation, and notes
   - Submit to add to your collection

#### Managing Your Collection

**Stats Overview**: View at a glance:
- Total objects captured
- Objects with photos vs without
- Total photos in your collection
- Number of different object types

**Filtering & Sorting**:
- **Search**: Find objects by name, type, or constellation
- **Filter by Type**: Galaxy, Nebula, Star Cluster, etc.
- **Filter by Photos**: Show only objects with/without photos
- **Sort by**: Name, Type, Date Added, or Photo Count
- **Sort Order**: Ascending or Descending

#### Adding Photos to Objects

Click on any object card to open the detail view:

1. **Add Photos**: Click "📷 Add Photo" button
2. **Upload Image**: Select an image file (PNG, JPG, JPEG, GIF, WebP)
3. **Add Metadata**:
   - Observation date
   - Exposition time (e.g., "120x30s")
   - Device/telescope used
   - Filters (e.g., "LRGB", "Ha-OIII")
   - ISO setting
   - Number of frames
   - Additional notes
4. **Submit**: Photo is added to the object

**Managing Photos**:
- **Set Main Photo**: Click ⭐ to set a photo as the main display image
- **Edit Photo**: Click ✏️ to update photo metadata
- **Delete Photo**: Click 🗑️ to remove a photo
- The first photo added is automatically set as the main photo
- If no photo exists, a default placeholder image is shown

#### Object Details

Each object shows:
- Main photo (or default placeholder)
- Object type and classification
- Source catalogue
- Constellation
- Coordinates (RA/Dec)
- Magnitude and size (if available)
- Personal notes
- Gallery of all photos

## 📖 Various

### 1. View System Logs

Navigate to **Configuration -> Logs** tab to:
- View real-time system logs
- Filter by log level (INFO, WARNING, ERROR)
- Monitor SkyTonight executions
- Debug issues

### 2. Automated SkyTonight execution

The system automatically:
1. Runs **twice a day**
   - 1 hour after last astro night
   - 1 hour before next astro night 
2. For each catalogue:
   - Generates tables of targets (DSO / Comets / Bodies)
   - Generates the interactive sky plot
   - Generates an altitude-vs-time graph for each target
