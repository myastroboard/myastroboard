## Key data flows

```
Browser request
  └─ app.py (Flask)
       ├─ Authentication check (auth.py)
       ├─ Cache hit? → Return cached JSON (cache_store.py)
       └─ Cache miss (202 Pending) → wait for background scheduler
                 ↑
       cache_scheduler.py (background daemon)
         └─ cache_updater.py (selective, per-job TTL)
               ├─ moon_phases.py, sun_phases.py, aurora_predictions.py…
               └─ weather_openmeteo.py, seeing_forecast_7timer.py…

SkyTonight pipeline (separate scheduler)
  skytonight_scheduler.py
    └─ skytonight_calculator.py
          ├─ skytonight_catalogue_builder.py (DSO dataset)
          ├─ skytonight_bodies.py (planets)
          ├─ skytonight_comets.py (comets)
          └─ Writes JSON → data/skytonight/calculations/
```

## Notes

- SkyTonight is the active target pipeline and scheduler path.
- Dataset generation is handled by `scripts/build_skytonight_catalogue.py`.
- All per-user data (Astrodex, equipment, plans) is stored in `data/` under the user's UUID.
- Never delete `data/secret_key.txt` or `data/vapid.json` — they cannot be regenerated without side effects.
