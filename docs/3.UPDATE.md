# Update

This guide covers common update workflows for MyAstroBoard.

## Update Published Image

```bash
docker compose pull --policy always
docker compose up -d --force-recreate
docker image prune -f
```

Check running version:

```bash
docker exec myastroboard cat /app/VERSION
```

## Update Local Development Build

```bash
git pull
docker compose down
docker compose up -d --build
docker image prune -f
```

## Backup Before Updating

Data is persisted in `data/`, but a backup is still recommended:

```bash
cp -r data data.backup
```

## Rollback to a Previous Version

```bash
docker compose down
docker pull worldofgz/myastroboard:v1.0.0
docker compose up -d
```

## Application Version

MyAstroBoard release version is managed in the root `VERSION` file.
Update this only after validating compatibility with application behavior,
API outputs, and scheduler expectations.

## Catalogue Data Pipeline

MyAstroBoard now uses the SkyTonight/PyOngc-based catalogue pipeline.

When catalogue behavior changes upstream, refresh generated data with:

```bash
python scripts/build_skytonight_catalogue.py
```

Then commit updated generated artefacts if they are tracked.
