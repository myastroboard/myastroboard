# 📖 Installation Guide

This guide covers all methods of installing and deploying MyAstroBoard.

## Prerequisites

### Required
- **Docker**
- **Docker Compose**
- **Internet connection** (for downloading images and target catalogues)

## Installation

### Option 1: Using Published Docker Image (Recommended for Users)

1. Create and move in the directory you want install this image
```bash
# Example
mkdir /opt/myastroboard
cd /opt/myastroboard
```

2. **Download the docker-compose file**:
```bash
curl -o docker-compose.yml https://raw.githubusercontent.com/WorldOfGZ/myastroboard/main/docker-compose.yml
```

3. **Start the application**:
```bash
docker compose up -d
```

4. **Access the dashboard**:
Open your browser and navigate to `http://localhost:5000` or IP of your machine.

The system will automatically:
- Start the web dashboard
- Initialize the scheduler
- Run SkyTonight & cache at dedicated time

5. **Update to latest version - Not necessary at first install**:
```bash
docker compose pull
docker compose up -d
```

### Option 2: Build from Source (For Developers)

1. **Clone the repository**:
```bash
git clone https://github.com/WorldOfGZ/myastroboard.git
cd myastroboard
```

2. **Edit docker-compose.yml file (if necessary)**:
```bash
nano docker compose.yml
```

3. **Build the application**:
```bash
docker compose build
```

4. **Start the application**:
```bash
docker compose up -d
```

5. **Access the dashboard**:
Open your browser and navigate to `http://localhost:5000`

The system will automatically:
- Start the web dashboard
- Initialize the scheduler
- Run SkyTonight & cache at dedicated time

6. **Debug & venv**
```bash
# Create venv
python -m venv venv

# Activate venv
# Windows
.\venv\Scripts\activate
# Linux/Mac
source venv/bin/activate

# Install requirements
pip install -r requirements.txt

# Install dev tools
pip install black ruff pytest pytest-cov
```

```bash
# Debug Launch
docker compose -f docker-compose-dev.yml -f docker-compose.debug.yml up --build
```

7. **Tests**

```bash
# Execute tests with covering
pytest --cov=backend tests/

# Execute specific test
pytest tests/test_target_loader.py -v
```

## Advanced configuration

### Security Configuration (IMPORTANT!)

For production deployments, you **must** configure a secure SECRET_KEY for session management:

1. **Generate a secure SECRET_KEY**:

   Choose one of these methods:
   
   ```bash
   # Method 1 - Python (recommended)
   python3 -c "import secrets; print(secrets.token_hex(32))"
   ```
   
   ```bash
   # Method 2 - OpenSSL
   openssl rand -hex 32
   ```
   
   Or use an online generator: [https://randomkeygen.com/](https://randomkeygen.com/) (use "CodeIgniter Encryption Keys")

2. **Set the SECRET_KEY**:

   Option A - Using .env file (recommended):
   ```bash
   # Copy the example file
   cp .env.example .env
   
   # Edit .env and set SECRET_KEY
   nano .env
   ```
   
   Option B - Directly in docker-compose.yml:
   ```yaml
   environment:
     - SECRET_KEY=your-generated-secret-key-here
   ```

**⚠️ WARNING**: If SECRET_KEY is not set, sessions will be invalidated on every container restart, forcing all users to log in again.

### HTTPS/SSL Configuration (Production)

For production deployments over HTTPS, enable secure cookies:

```yaml
environment:
  - SESSION_COOKIE_SECURE=true  # Only send cookies over HTTPS
```

**⚠️ IMPORTANT**: 
- Set `SESSION_COOKIE_SECURE=true` **only** when using HTTPS/SSL
- If enabled without HTTPS, users cannot log in (cookies won't be sent)
- For local/development deployments over HTTP, keep it `false` (default)

### Push Notifications - VAPID Contact Email

Background push notifications (including iOS) require a contact email address in the VAPID authentication token. This is sent to push services (Apple APNs, Google FCM) so they can reach you if there are delivery issues - it is **never shown to users**.

Set `VAPID_CONTACT_EMAIL` in `docker-compose.yml`:

```yaml
environment:
  - VAPID_CONTACT_EMAIL=your-email@example.com
```

**Notes:**
- The email domain does **not** need to match your app's domain - any real email address works.
- Without this setting, the app falls back to `admin@localhost`, which Apple APNs will reject, silently dropping all iOS push notifications.
- After setting this variable, restart the container. Existing subscriptions remain valid (VAPID keys are unchanged).

### iOS - Push Notifications Requirement

Push notifications on iOS require the app to be **installed as a PWA** (Progressive Web App):

1. Open the app in **Safari** on your iPhone or iPad.
2. Tap the **Share** button (square with arrow).
3. Select **"Add to Home Screen"**.
4. Launch the app from the Home Screen icon.

Push notifications will **not** work when opening the app directly in a Safari browser tab - they only work from the installed Home Screen shortcut. This is an Apple platform restriction.

The in-app test notification (Settings → Notifications → Test) fires a foreground notification and works in any tab - this is **not** a push notification and does not confirm that background push delivery works.

### Remember Me Feature

The login page includes a "Remember me" checkbox (only visible when `SESSION_COOKIE_SECURE=true`):
- When **checked**: Keeps users logged in for 30 days
- When **unchecked**: Session expires when browser closes
- **Availability**: Only shown when using HTTPS (SESSION_COOKIE_SECURE=true) for security
- **HTTP mode**: Feature is hidden; sessions always expire when browser closes
- Logout always clears all cookies and sessions

### Default Credentials

On first run, a default admin account is created:
- **Username**: `admin`
- **Password**: `admin`

**🔒 IMPORTANT**: Change the default password immediately after first login! The application will display a warning banner until you do.

### Environment Variables

Key configuration options (set in `docker-compose.yml`):

```bash
# Security
- SECRET_KEY=your-secret-key-here-replace-with-random-64-character-hex-string
- SESSION_COOKIE_SECURE=false  # Set to "true" only when using HTTPS

# Push notifications
- VAPID_CONTACT_EMAIL=your-email@example.com  # Required for iOS push notifications

# Directories (defaults work for Docker)
# App
- DATA_DIR=/app/data

# Reverse Proxy Configuration (NGINX Proxy Manager, Traefik, etc.)
# CRITICAL: Set TRUST_PROXY_HEADERS=true when using a reverse proxy with HTTPS termination
# This allows Flask to trust X-Forwarded-* headers from the proxy
# See docs/6.REVERSE_PROXY.md for complete setup guide
- TRUST_PROXY_HEADERS=true
```

### Data Persistence

All configuration and outputs are stored in Docker volumes:
- `./data/` - Configuration files (survives container rebuilds, with some exception)
- `./data/astrodex/` - Astrodex main folder
- `./data/cache/` - Main cache file (will be removed at docker up)
- `./data/equipments/` - Equipments main folder
- `./data/projects/` - Plan my Night main folder
- `./data/skytonight/` - SkyTonight main folder
  - `./data/skytonight/calculations/` - Calculations generated twice a day (will be removed at docker up)
  - `./data/skytonight/catalogues/` - Generated SkyTonight catalogues
  - `./data/skytonight/logs/` - Skytonight logs (will be removed at docker up)
  - `./data/skytonight/output/` - Graph outputs (will be removed at docker up)
  - `./data/skytonight/runtime/` - Runtime files for scheduler (will be removed at docker up)

**Important**: These directories persist across Docker container rebuilds, ensuring your settings and data are never lost.
