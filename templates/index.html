    - Illumination Sun < -18° AND Moon illumination < 15% => Visible moon, but faint

        All parameters of this page are used by SkyTonight internals.

        Altitude constraints

        Targets must reach an altitude between min and max during the observable night. The green band is the valid zone.

        Airmass

        Airmass measures the thickness of atmosphere a light ray crosses. Lower altitude = more atmosphere = worse quality. This constraint overrides min altitude if stricter.

        Angular size (arcmin)

        Filters deep-sky objects by apparent size. Use to remove objects too small to resolve or too large for your sensor's field of view.

        Moon separation

        Targets within the exclusion zone around the moon are filtered out. With dynamic mode enabled, the minimum gap automatically scales with moon brightness.

        Observable time threshold

        A target must be above all constraints for at least this fraction of the night. Lower values include objects visible only briefly near dusk or dawn.

        Backup

        Download a ZIP archive of your personal data. The archive includes configuration, user accounts, Astrodex entries and equipment profiles.

        Included in the archive:

        • data/config.json
        • data/users.json
        • data/astrodex/
        • data/equipments/

        Restore

        Restoring a backup will overwrite the current configuration, users, Astrodex and equipment data with the contents of the archive. This action cannot be undone.

        Current Log Level

        The log level controls how much detail is written to the log files. To capture the most detail for troubleshooting, set the level to DEBUG via the LOG_LEVEL environment variable, restart the application, reproduce the issue, then export the logs below.
        Active file log level: -

        LOG_LEVEL=DEBUG  ·  Set in docker-compose.yml or .env, then restart the container.
